# Web RTC 系统检测工具

基于火山引擎 Web RTC SDK 的系统兼容性检测工具。

## 功能特性

- **浏览器兼容性检测**：检测浏览器是否支持 RTC 相关 API
- **设备获取能力检测**：枚举麦克风、摄像头和扬声器设备
- **视频编码检测**：检测 H264、H265、VP8 编码支持情况
- **扬声器播放检测**：播放测试音验证音频输出功能
- **分辨率支持检测**：检测设备支持的视频分辨率
- **协议支持检测**：检测 SDP、SRTP、UDP、TCP 协议支持情况

## 协议检测方法说明

### SDP 协议（会话描述协议）
- **检测方法**：检查 `RTCPeerConnection.createOffer` 方法是否存在
- **原理**：createOffer 是生成 SDP 的核心方法，存在即表示支持 SDP 协议

### SRTP 协议（安全实时传输协议）
- **检测方法**：检查 `RTCPeerConnection.setConfiguration` 方法是否存在
- **原理**：setConfiguration 可以用来配置加密参数，支持该方法通常意味着浏览器支持 SRTP

### UDP 协议（用户数据报协议）
- **检测方法**：默认支持
- **说明**：WebRTC 通过 ICE 框架自动选择传输协议，现代浏览器都支持 UDP 作为首选传输协议

### TCP 协议（传输控制协议）
- **检测方法**：默认支持
- **说明**：WebRTC 也支持 TCP 作为备用传输协议（当 UDP 不可用时），现代浏览器都支持

## 快速开始

### 方法一：使用 Python 服务器（推荐）

```bash
python3 -m http.server 8080
```

然后在浏览器中访问：http://localhost:8080

### 方法二：使用 npm 服务

```bash
npm install
npm run serve
```

然后在浏览器中访问：http://localhost:8080

## 使用说明

1. 打开页面后点击「开始检测」按钮
2. 允许浏览器访问摄像头和麦克风权限
3. 查看各项检测结果
4. 底部日志区域显示详细的检测过程

## 注意事项

- 必须使用 HTTPS 或 localhost 访问，否则无法获取媒体设备权限
- 检测过程中请确保设备已正确连接
- 扬声器检测会播放一段测试音，请确保音量适中
